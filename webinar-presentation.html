Lessons Learned

API Traps That
Cost Me 2 Days

Things the Swagger docs don't tell you

Trap #1

metadata.message, NOT body

What you'd expect

variants: [{
  subject: "...",
  body: "<p>...</p>"
}]

Returns 200 OK. Email body is empty. Silent failure.

What actually works

variants: [{
  metadata: {
    subject: "...",
    message: "<p>...</p>"
  }
}]

Body = metadata.message. Not body. Not content.

Trap #2

IDs Are Integers, Not Strings

FieldWrongRight
actionId"123"123
branchId"456"456
senderProfileIds["1", "2"][1, 2]

Strings return 400 or get silently ignored.

Traps #3-5

Trap 3: waitDays vs wait_in_minutes

Create: waitDays: 3 (integer, days)
Update: wait_in_minutes: 4320 (integer, minutes)

Trap 4: Two APIs, Two Pagination Styles

Core API: limit + offset
Multichannel API: page + limit (page starts at 1)

Trap 5: Schedule Disabled Days

"from": 0, "to": 0 → 422 error
"from": 0, "to": 23, "enabled": false → works
